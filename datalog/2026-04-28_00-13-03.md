## 项目深入分析报告

### 1. 项目概述

**BZPlayer** 是一个跨平台的视频播放器项目，采用了**双前端**架构：

| 组件 | 技术栈 | 位置 |
|------|--------|------|
| Web 前端 | 原生 HTML5 Video + JavaScript | `/index.html` |
| macOS 原生客户端 | SwiftUI + AppKit + libmpv | `/macos/BZPlayer/` |

**核心定位**：最高支持 **16x 倍速播放**的轻量级播放器，聚焦倍速播放体验，底层使用 `mpv` 保障格式兼容性。

---

### 2. 技术架构

#### 2.1 macOS 客户端（主要开发方向）

```
BZPlayer (Package.swift)
├── BZPlayerApp (executable)
│   ├── BZPlayerApp.swift         # App 入口、AppDelegate、设置面板
│   ├── PlayerViewModel.swift     # 1978 行，核心状态管理与业务逻辑
│   ├── PlayerContainerView.swift # SwiftUI ↔ AppKit 桥接
│   ├── PlayerRootView.swift      # SwiftUI 视图层
│   └── MpvPlayer.swift           # 654 行，libmpv C 绑定与渲染回调
└── CMpv (systemLibrary)
    ├── module.modulemap           # 公开 libmpv C API 给 Swift
    └── shim.h                     # mpv 头文件的薄封装
```

**关键技术决策**：
- **libmpv** 而非 AVPlayer → 更好的格式兼容性和播放控制
- **OpenGL** 渲染管线 → 通过 `mpvRenderUpdateCallback` + `mpvOpenGLGetProcAddress` 接入
- **SwiftUI + AppKit 混用** → `PlayerContainerView` (NSViewRepresentable) 做桥接
- **多窗口支持** → 通过 `appDelegate` 管理多个 `PlayerViewModel` 实例
- **ffprobe 媒体信息** → `probeMediaInfo()` 调用外部 ffprobe 获取流信息

#### 2.2 Web 前端

纯原生实现，无框架依赖：
- HTML5 `<video>` 标签
- 手动实现倍速按钮（0.25x → 16x 共 8 档 + ±0.25x 微调）
- 点击/双击交互（暂停、全屏）
- 进度条拖拽

### 3. 核心功能实现

| 功能 | 实现位置 | 说明 |
|------|----------|------|
| 倍速播放 (0.25x-16x) | `PlayerViewModel` + UI 按钮 | 支持 ±0.25x 微调 |
| 点击暂停/双击全屏 | `PlayerHostView` (ClickView) | 单击/双击区分 |

### 5. 值得注意的代码特征

1. **大量调试日志**: 每个 Swift 文件都挂了一个 `debugLog()` 函数写往 `~/debug_bzplayer.log`
2. **文件极大**: `PlayerViewModel.swift` 达 1978 行，`BZPlayerApp.swift` 达 708 行，缺少文件拆分
3. **C ↔ Swift 桥接**: `CMpv` 系统库通过 `module.modulemap` 暴露 libmpv 的 C API
4. **OpenGL 渲染**: 使用 `dlopen` 手动加载 OpenGL framework，而非 Swift 的 `OpenGL` 模块导入
5. **Window 访问器模式**: `WindowAccessor` + `WindowAccessorView` 解决 SwiftUI 无法直接获取 `NSWindow` 的问题

### 6. 主要问题与风险
